## Summary

Important approved review findings should become normal AgentX issues so they can be prioritized,
sequenced, and completed through the same backlog lifecycle as other work.

## Guidance

- Promote findings marked `recommended` or `required` when they need work beyond the current review cycle.
- Carry priority into issue labels and preserve dependencies and evidence links in the issue body.
- Link the created backlog issue back into the durable finding record to avoid duplicates.

## Use When

- Defining review-finding promotion rules.
- Building tools that convert findings into tracked follow-up work.
- Explaining when a finding should remain review-only versus become backlog work.

## Avoid

- Leaving important deferred findings only in review prose.
- Creating separate tracking records that are disconnected from the AgentX issue flow.
