# Power BI Source Control Strategy

Guidelines for storing and managing Power BI report and semantic model files in GitHub Enterprise.

## Format: PBIP (Required)

All Power BI artifacts committed to this repository MUST use **PBIP format** (Power BI Project).

PBIP stores reports as a folder of JSON and TMDL files that produce clean, human-readable git diffs. `.pbix` binary files cannot be diffed, reviewed, or merged in git.

```
reports/
+-- SalesReport.Report/          <- Report layer (visuals, layout, pages)
|   +-- report.json
|   +-- StaticResources/
|       +-- SharedResources/
|           +-- BaseThemes/
|               +-- PwCTheme.json
+-- SalesReport.SemanticModel/   <- Or in models/ if shared across reports
    +-- definition/
    |   +-- model.tmdl
    |   +-- tables/
    |   |   +-- fact_sales.tmdl
    |   |   +-- dim_date.tmdl
    |   |   +-- dim_product.tmdl
    |   +-- relationships.tmdl
    |   +-- roles/
    |       +-- RowLevelSecurity.tmdl
    +-- .pbi/                    <- GITIGNORED - local developer settings
        +-- localSettings.json
```

## .gitignore Additions

Add the following to `.gitignore` for all Power BI repositories:

```gitignore
# Power BI local settings (developer-specific, not for source control)
**/.pbi/localSettings.json
**/.pbi/

# Compiled output (generated by CI, not source)
dist/*.pbix
dist/

# Temporary Power BI Desktop files
*.pbix.lock
~$*.pbix

# DAX Studio and Tabular Editor working files
*.daxx
*.te3backup
```

## Folder Structure

```
<repo-root>/
+-- reports/                     # PBIP report folders (one subfolder per report)
|   +-- {ReportName}.Report/
|   +-- {ReportName}.SemanticModel/   <- if report-specific model
+-- models/                      # Shared semantic models (used by multiple reports)
|   +-- {ModelName}.SemanticModel/
+-- docs/
|   +-- powerbi/
|       +-- TOOLCHAIN.md
|       +-- SOURCE-CONTROL.md
|       +-- RLS-MATRIX.md        <- Created when RLS is configured
|       +-- WORKSPACES.md        <- Workspace GUIDs per environment
+-- scripts/
|   +-- deploy/
|       +-- powerbi/             <- PowerShell deployment scripts
+-- dist/                        <- Gitignored: compiled .pbix output from CI
```

## Branching Strategy

Mirrors the Power BI deployment pipeline stages (Dev -> Test -> Prod).

```
main          <- Corresponds to Production workspace
  ^
release/*     <- Corresponds to Test workspace (e.g. release/2026-q1)
  ^
develop       <- Corresponds to Dev workspace
  ^
feature/*     <- Developer branches (e.g. feature/sales-report-rls)
```

### Branch Naming

```
feature/{report-name}           <- New report or major change
fix/{report-name}-{issue}       <- Bug fix on existing report
chore/{task-description}        <- Non-functional changes (theme, docs)
```

### Pull Request Flow

```
feature/sales-report-rls
  -> PR to develop
     CI: powerbi-validate.yml (pbi-tools compile + BPA check)
     Review: Code Reviewer (DAX quality, PBIP compliance)
  -> Merged to develop
     CI: powerbi-deploy.yml (publish to Dev workspace)
     CI: powerbi-refresh.yml (trigger Dev refresh)
  -> PR to release/2026-q1
     Promoted to Test workspace via Fabric Deployment Pipeline
  -> PR to main
     Promoted to Production workspace via Fabric Deployment Pipeline
```

## Migrating Existing .pbix Files

For any legacy `.pbix` files that need to be brought into source control:

```powershell
# Step 1: Extract .pbix to PBIP format
pbi-tools extract `
    -pbixPath "legacy/SalesReport.pbix" `
    -extractFolder "reports/SalesReport"

# Step 2: Review extracted files
git status  # Shows all new PBIP files

# Step 3: Add to .gitignore (if keeping .pbix separately for reference)
echo "legacy/*.pbix" >> .gitignore

# Step 4: Commit PBIP files
git add reports/SalesReport/
git commit -m "chore: migrate SalesReport.pbix to PBIP format (#issue)"
```

For very large historical `.pbix` files that must be retained:
```
# Option: Git LFS for legacy binaries only
git lfs track "legacy/*.pbix"
git add .gitattributes
```

## Code Review Standards for Power BI PRs

Reviewers MUST check (in addition to general code review):

**DAX Measures:**
- [ ] VAR/RETURN used for multi-step measures
- [ ] DIVIDE() used instead of `/` operator
- [ ] No visual-level measures (all in semantic model)
- [ ] Display folders populated

**Power Query:**
- [ ] Explicit column types on every query
- [ ] Unused columns removed early in pipeline
- [ ] Query folding verified for large table filters (comment in PR)
- [ ] No hardcoded server names (use parameters)

**PBIP Compliance:**
- [ ] No .pbix binary files in `reports/` or `models/`
- [ ] pbi-tools compile output in PR CI check = green
- [ ] `.pbi/localSettings.json` not committed

**Security:**
- [ ] No credentials/connection strings in report files
- [ ] RLS documented in `docs/powerbi/RLS-MATRIX.md`
- [ ] RLS tested with both pass and fail user scenarios
